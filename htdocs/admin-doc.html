Loudwater Content Management System
Administrative Interface
 
 
Edit Server Data Screen
 
HOSTNAME   The hostname parameter is the fully-qualified domain name of the server to which this configuration record refers.
 
 
IP ADDRESS   The ip address parameter is the IPv4 address of the server, in dotted-quad notation.
 
 
TOTAL THREADS   The total threads parameter is the absolute limit on total number of simultaneous work threads (ingest, encode, distribution and maintenance) that lwpd(8) will allow on this server.
 
 
INGEST THREADS   The ingest threads parameter is the maximum number of simultaneous ingest threads that lwpd(8) will allow on this server.
 
 
ENCODE THREADS   The encode threads parameter is the maximum number of simultaneous encode threads that lwpd(8) will allow on this server.
 
 
DISTRIBUTION THREADS   The distribution threads parameter is the maximum number of simultaneous distribution threads that lwpd(8) will allow on this server.
 
 
MAINTENANCE THREADS   The maintenance threads parameter is the maximum number of simultaneous maintenance threads that lwpd(8) will allow on this server.
 
 
 
 
Add Player Screen
 
PLAYER NAME   The player name parameter uniquely identifies a partcular player configuration. It is passed as the 'name=' parameter in the URL when invoking the player.
 
 
 
Edit Player Screen
 
PLAYER NAME   The player name is a read-only parameter. It uniquely identifies a partcular player configuration and is passed as the 'name=' parameter in the URL when invoking the player.
 
 
TITLE   The title parameter determines the text that will appear on the title bar of the player.
 
 
LAYOUT   The layout parameter determines basic player layout and video screen width. Two layouts are available:
  • Standard - Optimized for use with video with a 4x3 aspect ratio.
  • Wide-Screen - Optimized for use with video with a 16x9 aspect ratio.
    •  
     
    PLAYLIST POSITION   The playlist position parameter determines where the playlist will be positioned relative to the video display. Possible options are:
  • Left - Display playlist to the left of the video area.
  • Right - Display playlist to the right of the video area.
  • Below - Display playlist below the video area.
  • Over - Display playlist in the same region as the video area, hiding it when media is actually playing.
  • None - Do not display a playlist at all.
    •  
     
    PLAYLIST TEXT COLOR   The playlist text color parameter determines the color that will be used in the playlist text of the player. This should be a web RGB value (e.g. "#123456").
     
     
    PLAYLIST BACKGROUND COLOR   The playlist background color parameter determines the color that will be used in the playlist background of the player. This should be a web RGB value (e.g. "#123456").
     
     
    PLAYLIST HIGHLIGHT COLOR   The playlist highlight color parameter determines the highlight color that will be used in the playlist text of player when the user floats the mouse over an entry. This should be a web RGB value (e.g. "#123456").
     
     
    SCREEN BACKGROUND COLOR   The screen background color parameter determines the color that will be used in the background of the video screen. This should be a web RGB value (e.g. "#123456").
     
     
    OVERALL BACKGROUND COLOR   The overall background color parameter determines the color that will be used in the overall background of the player. This should be a web RGB value (e.g. "#123456").
     
     
    SPLASH LINK   The splash link parameter is a URL that points to an FLV video that the player will run once when it is first initialized. If no splash video is desired, then this field should be left blank.
     
     
    AUDIO LOGO LINK   The audio logo link parameter is a URL that points to an image or video to be displayed during the playing of on-demand audio-only content.
     
     
    VIDEO LOGO LINK   The video logo link parameter is a URL that points to an image or video to be displayed during the playing of on-demand video content.
     
     
    CUSTOM BRANDING LINK  
    The custom branding link parameter is a URL that points to the directory that contains the custom branding content for this player.

    Branding content is invoked by passing a 'brandid=<id>' argument when spawning the player or banner. The following custom-branded elements are loaded, overriding the elements specified in the player's base configuration:

     
    FILENAMECONTENT
    audiologo_<id>.jpg Audio Logo
    videologo_<id>.jpg Video Logo
    hbannerlogo_<id>.png Horzontal Banner Logo
    hbannertext_<id>.png Horzontal Banner Text Background
    vbannerlogo_<id>.png Vertical Banner Logo
    vbannertext_<id>.png Vertical Banner Text Background
    gateway_<id>.flv Gateway Video
     
     
    LINK BUTTON CODE   The link button code parameter is currently unimplemented.
     
     
    BANNER HIMAGE URL   The banner himage url is a URL to the image to display in the "horizontal" style click-through banner for this player.
     
     
    BANNER HIMAGE WIDTH   The banner himage width is the width (in pixels) to apply to the image in the "horizontal" style click-through banner for this player.
     
     
    BANNER VIMAGE URL   The banner vimage url is a URL to the image to display in the "vertical" style click-through banner for this player.
     
     
    BANNER VIMAGE HEIGHT   The banner vimage height is the height (in pixels) to apply to the image in the "vertical" style click-through banner for this player.
     
     
    DEFAULT LINK   The default link parameter is a URL that points to playlist to be displayed when a live stream is requested at a time when no live stream is active.
     
     
    LIVE SEGMENTS   The live segments section lists the currently configured live segments for this player. Segments can be added, deleted or changed by clicking the Edit Live Segment button at the bottom of the Edit Player screen.
     
     
    ANDO STATION ID   The ando station id parameter is an integer value used to associate this player to a particular "station" defined in the Ando Ad Injection system. If no Ando station is desired, enter a "0" for this parameter.
     
     
    AD QUALITY   The ad quality parameter is an integer value used to indicate the quality/bandwidth of the ad creatives inserted by Ando. Legal values are:
  • 1 = Low (32 kbit/sec typical)
  • 2 = Medium (64 kbit/sec typical)
  • 3 = High
    •  
     
    USE SYNCHED BANNERS   The use synched banners parameter is a boolean value used to indicate whether or not synched banners supplied by Ando should be displayed in the player. The use of synched banners is incompatible with certain other ad serving systems (e.g. Google AdSense).
     
     
    BUTTON SECTION IMAGE LINK   The button section image link parameter is a URL that points to an image to be displayed in place of the Channel Button section of the player. The dimensions of the image should be 296x128. NOTE: this image will be displayed only if the BUTTON COLUMNS and BUTTON ROWS parameters are "0".
     
     
    BUTTON COLUMNS   The button columns parameter is an integer that indicates the number of columns of buttons to display in the Channel Button section of the player.
     
     
    BUTTON ROWS   The button rows parameter is an integer that indicates the number of rows of buttons to display in the Channel Button section of the player.
     
     
    HEAD CODE   The head code parameter is arbitrary HTML code to be included in the head section of the player. A typical use would be to include scripts needed to operate with third-party ad management systems (e.g. Google AdSense).
     
     
    TOP BANNER CODE   The top banner code parameter is HTML code for displaying and linking to the banner to be displayed in the top banner position of the player.
     
     
    SIDE BANNER CODE   The side banner code parameter is HTML code for displaying and linking to the banner to be displayed in the side banner position of the player.
     
     
    SOCIAL TEST   The social test parameter enables display of the icon for the 'test' social media service in the player. [NOTE: this parameter is for testing only, and should never be used on a production player!]
     
     
    SOCIAL DISPLAY LINK   The social display link parameter enables display of the direct URL for the currently selected episode in the player.
     
     
    SOCIAL FACEBOOK   The social facebook parameter enables display of the Like button for the FaceBook social media service in the player. Enabling this parameter will also require that the social facebook admin parameter (see below) be configured.
     
     
    SOCIAL FACEBOOK ADMIN   The social facebook admin parameter is used to specify the FaceBook administrator or application ID for this page.
     
     
     
     
    Edit Channel Screen
     
    NAME   The name parameter is a read-only field that contains the name of the channel.
     
     
    TITLE   The title parameter is a text field. This value will appear by default in the TITLE field in new posts.
     
     
    DESCRIPTION   The description parameter is a multiline text field. This value will appear by default in the DESCRIPTION field in the media's metadata.
     
     
    CATEGORY   The category parameter is a text field. This value will appear by default in the CATEGORY field in new posts.
     
     
    LINK   The link parameter is a URL, and should point to a page where more information regarding the referenced content can be found. This value will appear by default in the LINK field in new posts.
     
     
    COPYRIGHT   The copyright parameter is a text field and should contain the year and owner name for a copyright notice for the referenced content (e.g. "2009 Big Company"). This value will appear by default in the COPYRIGHT field in new posts.
     
     
    WEBMASTER   The webmaster parameter is a text field and should contain the e-mail address for the editor responsible for the referenced content (e.g. "editor@example.com"). This value will appear by default in the WEBMASTER field in new posts.
     
     
    AUTHOR   The author parameter is a text field and should contain the name of the author of the referenced content. This value will appear by default in the AUTHOR field in new posts.
     
     
    OWNER   The owner parameter is a text field and should contain the name of the owner of the referenced content.
     
     
    OWNER_EMAIL   The owner email parameter is a text field and should contain the e-mail address for the owner of the referenced content (e.g. "owner@example.com").
     
     
    SUBTITLE   The subtitle parameter is a text field and should contain the subtitle for the referenced channel.
     
     
    ITUNES CATEGORY   The itunes category parameter is a text field and should contain the category fields for the iTunes aggregator site for the referenced channel.
     
     
    KEYWORDS   The keywords parameter is a text field and should contain the keywords for the referenced channel.
     
     
    EXPLICIT   The explicit parameter is a drop-down menu with three choices: Yes, to indicate that the channel contains explicit content; Clean, to indicate that the channel contains no explicit content, and No, to provide no indication as to the nature of the channel contents. This value will appear by default in the EXPLICIT field in new posts.
     
     
    LANGUAGE   The language parameter is a text field and should contain the ISO language code for the language used in the referenced content (e.g. American English would use "en_us"). This value will appear by default in the LANGUAGE field in new posts.
     
     
    MAXIMUM UPLOAD SIZE   The maximum upload size parameter is the maximum file length (measured in bytes) that will be accepted for upload in this channel.
     
     
    THUMBNAIL UPLOAD URL   The thumbnail upload url parameter is a URL that points to the remote directory to be used to serve thumbnail images for this channel. The protocol used must be one that is capable of uploading content (e.g. "ftp://").
     
     
    THUMBNAIL UPLOAD USERNAME   The thumbnail upload username parameter is username for the account used to authenticate thumbnail image uploads to the server.
     
     
    THUMBNAIL UPLOAD PASSWORD   The thumbnail upload password parameter is the password associated with the thumbnail upload username parameter (above) for authenticating thumbnail image uploads to the server.
     
     
    THUMBNAIL DOWNLOAD URL   The thumbnail download url parameter is a URL that points to the remote directory to be used to serve thumbnail images for this channel. This value will form the basis of download URL served to clients.
     
     
    THUMBNAIL DOWNLOAD URL   The thumbnail download url parameter is a URL that points to the remote directory to be used to serve thumbnail images for this channel. The protocol used must be one that is capable of downloading content (e.g. "http://").
     
     
    DEFAULT THUMBNAIL   The default thumbnail parameter specifys the default thumbnail image that will be used for content on this channel.
     
     
    DOWNLOAD PREAMBLE   The download preamble parameter is a URL that is inserted into the download URL generated to clients. It can be used to direct downloads through third-party traffic measurements systems.
     
     
    CHANNEL TAPS   The channel taps list show the set of configured taps for the channel.
     
     
     
     
    Edit Live Segment Screen
     
    LIVE LINK   The live link parameter is a URL that points to the live stream to be played during the defined days/times.
     
     
    LOGO LINK   The logo link parameter is a URL that points to an image or video to be displayed during playout of the associated live stream.
     
     
    ANDO STATION ID   The ando station id parameter is an integer value used to associate this stream to a particular "station" defined in the Ando Ad Injection system. If no Ando station is desired, enter a "0" for this parameter.
     
     
    START TIME   The start time parameter is the time at which live playout for the segment begins. This value should be in Universal Coordinated Time (UTC) (formerly known as "Greenwich Mean Time").
     
     
    RUN HOURS   The run hours parameter is the number of hours following the START TIME that the segment will be active.
     
     
    RUN ON   The run on section indicates which day(s) of the week the segment will be active on. The day of the week will be calculated on the basis of Coordinated Universal Time (formerly known as "Greenwich Mean Time").
     
     
     
     
    Edit Channel Button Screen
     
    IMAGE URL   The image url parameter is a URL that points to the image to display on the channel button when the channel is not selected.
     
     
    ACTIVE IMAGE URL   The active image url parameter is a URL that points to the image to display on the channel button when the channel is currently selected.
     
     
    CLICK-THROUGH URL   The click-through url parameter is a URL that points to the playlist to be loaded in the player when the button is clicked.
     
     
    ANDO STATION ID   The ando station id parameter is an integer value used to associate this channel to a particular "station" defined in the Ando Ad Injection system. If no Ando station is desired, enter a "0" for this parameter.
     
     
    BUTTON MODE   The button mode parameter is used to configure this button as either a live feed or on demand selector. When set for 'OnDemand Channel' (the default), clicking the button will cause the player to execute the click through url parameter, whereas a setting of 'Live Feed' will cause the appropriate configured live feed to be started.
     
     
     
     
    Edit Post Screen
     
    NAME   The name parameter is a read-only field that contains the name of the channel to which the post belongs.
     
     
    TITLE   The title parameter is a text field.
     
     
    DESCRIPTION   The description parameter is a multiline text field.
     
     
    CATEGORY   The category parameter is a text field.
     
     
    LINK   The link parameter is a URL, and should point to a page where more information regarding the referenced content can be found.
     
     
    COPYRIGHT   The copyright parameter is a text field and should contain the year and owner name for a copyright notice for the referenced content (e.g. "2009 Big Company").
     
     
    WEBMASTER   The webmaster parameter is a text field and should contain the e-mail address for the editor responsible for the referenced content (e.g. "editor@example.com").
     
     
    COMMENTS   The comments parameter is a text field and should contain audience comments for the referenced content.
     
     
    AIR DATE   The air date parameter is a text field that should contain the nominal date of the original airing of the broadcast, in format MM/DD/YYYY. For new posts, this value will default to the date of the post.
     
     
    AIR HOUR   The air hour parameter is an integer field that should contain the nominal hour of the original airing of the broadcast, starting with '1'.
     
     
    LANGUAGE   The language parameter is a text field and should contain the ISO language code for the language used in the referenced content (e.g. American English would use "en_us").
     
     
    ACTIVE   The active box indicates the current status of the post. When set to Yes, the post will be visible online, whereas setting the value to No will hide the post online.
     
     
    SEGMENTS   The segments parameter is a read-only value that indicates the number of segments that comprise the post.
     
     
    POSTED ON   The POSTED ON parameter is a read-only value that indicates the date and time that the post was uploaded.
     
     
    POSTED BY   The POSTED BY parameter is a read-only value that indicates the name of user that uploaded the post.
     
     
    POSTED FROM   The POSTED FROM parameter is a read-only value that indicates IP address of the system from which the post was uploaded.
     
     
    JOB STATUS   The JOB STATUS parameter is a read-only value indicates the current status of the post. The possible values are:
    Complete Processing is complete and the post visible online.
    Uploading The post is in the process of being uploaded.
    Queued for Ingest The post has been uploaded and is waiting for ingest processing.
    Ingesting The post is currently being ingested.
    Queued for Encode The post has been ingested and is waiting for encoding.
    Encoding The post is currently being encoded.
    Queued for Distribution The post has been encoded and is waiting for distribution to one or more tap site(s).
    Distributing The post is currently being distributed to one or more tap sites.
     
     
    Edit Tap Screen
     
    CHANNEL NAME   The channel name parameter is a read-only field that contains the name of the channel to which this tap belongs.
     
     
    TITLE   The title parameter is a text field which appears in listings of the taps for a given channel.
     
     
    CREATED ON   The created on parameter is a read-only value that gives the date and time when this tap was created.
     
     
    LAST MODIFIED ON   The last modified on parameter is a read-only value that gives the date and time when the configuration and/or content of the tap was last changed.
     
     
    VISIBILITY WINDOW   The visibility window parameter is an interger value that indicates for how many days past the air date should be visible on the tap listing. When this period is past, the underlying media will be available via direct linking.
     
     
    UPLOAD URL   The upload url parameter is a URL that points to the remote directory to be used to serve content for this tap. The protcol used must be one that is capable of uploading content (e.g. "ftp://").
     
     
    UPLOAD USERNAME   The upload username parameter is username for the account used to authenticate content uploads to the server.
     
     
    UPLOAD PASSWORD   The upload password parameter is the password associated with the upload username parameter (above) for authenticating content uploads to the server.
     
     
    DOWNLOAD URL   The download url parameter is a URL that points to the remote directory to be used to serve content for this tap. This value will form the basis of download URL served to clients.
     
     
    DOWNLOAD PREAMBLE   The download preamble parameter is a URL that is inserted into the download URL generated to clients. It can be used to direct downloads through third-party traffic measurements systems.
     
     
    PING URL   The ping url parameter is a URL that is executed each time the configuration or content of the tap changes.
     
     
    VALIDATION URL   The validation url parameter is a URL to be used when validating the feed by means of the 'Validate' button on the 'Links' page. If left blank, then no 'Validate' button will be displayed for the tap.
     
     
    TRANSPARENCY MODE   The transparency mode parameter is a boolean that, when set to 'yes', causes the raw input file uploaded to the channel to be sent directly to the specified tap location without any processing or conversion whatever. NOTE: this parameter is mutually exclusive with the audio encoder command, audio extension, video encoder command and video extension parameters.
     
     
    AUDIO ENCODER COMMAND   The audio encoder command parameter is the CLI command to be executed to encode audio-only content for this tap. The following wildcard characters are supported:
    WildcardValue
    %fInput filename
    %FOutput filename
     
     
    AUDIO EXTENSION   The audio extension parameter is the filename extension to use for files produced with the audio encoder command (e.g. "mp3", "ogg").
     
     
    AUDIO MIMETYPE   The audio mimetype parameter is the mimetype string to use when serving audio content on this tap.
     
     
    VIDEO ENCODER COMMAND   The video encoder command parameter is the CLI command to be executed to encode audio/video content for this tap. The following wildcard characters are supported:
    WildcardValue
    %fInput filename
    %FOutput filename
     
     
    VIDEO EXTENSION   The video extension parameter is the filename extension to use for files produced with the video encoder command (e.g. "avi", "flv").
     
     
    VIDEO MIMETYPE   The video mimetype parameter is the mimetype string to use when serving video content on this tap.
     
     
    FORCE AUDIO SAMPLERATE   The force audio samplerate parameter is the sample rate (in samples per second) to which audio should be converted before executing the encoder phase. If no sample rate is preferred, this field should be left blank.
     
     
    FORCE AUDIO CHANNELS   The force audio channels parameter is the number of channels to which audio should be converted before executing the encoder phase. If no channel count is preferred, this field should be left blank.
     
     
    HEADER TEMPLATE   The header template parameter is text which is prepended to the playlist data served for this tap.
     
     
    CHANNEL TEMPLATE   The channel template parameter is text which is included in the channel section of the playlist data served for this tap. The following wildcards are supported:
    WildcardValue
    %CHANNEL_NAME%The name parameter for the channel.
    %CHANNEL_TITLE%The title parameter for the channel.
    %CHANNEL_DESCRIPTION%The description parameter for the channel.
    %CHANNEL_CATEGORY%The category parameter for the channel.
    %CHANNEL_LINK%The link parameter for the channel.
    %CHANNEL_COPYRIGHT%The copyright parameter for the channel.
    %CHANNEL_WEBMASTER%The webmaster parameter for the channel.
    %CHANNEL_LANGUAGE%The language parameter for the channel.
    %TAP_URL%The URL of the referenced tap.
    %PUBLISH_DATE%The date/time that the channel was created.
    %BUILD_DATE%The date/time that the channel was last updated.
    %GENERATOR%Loudwater name and version number.
     
     
    ITEM TEMPLATE   The item template parameter is text which is included in the item section of the playlist data served for this tap. The following wildcards are supported:
    WildcardValue
    %ITEM_NAME%The name parameter for the item.
    %ITEM_TITLE%The title parameter for the item.
    %ITEM_DESCRIPTION%The description parameter for the item.
    %ITEM_CATEGORY%The category parameter for the item.
    %ITEM_LINK%The link parameter for the item.
    %ITEM_COPYRIGHT%The copyright parameter for the item.
    %ITEM_WEBMASTER%The webmaster parameter for the item.
    %ITEM_LANGUAGE%The language parameter for the item.
    %ITEM_PUBLISH_DATE%The date/time that the item was posted.
    %ITEM_CONTENT_URL%The URL to the media content.
    %ITEM_AIR_DATE%The nominal air date of the item, in RFC822 format.
    %ITEM_AIR_HOUR%The nominal air hour of the item.
    %ITEM_LAST_MODIFIED_DATETIME%The date/time that this item's metadata was last modified, in RFC822 format.
    %ITEM_CONTENT_MIMETYPE%The MIME type string to serve with the media content.
     
     
    FOOTER TEMPLATE   The footer template parameter is text which is appended to the playlist data served for this tap.
